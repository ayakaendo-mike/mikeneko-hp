# 要件定義

## 1. 確定事項サマリー

| 項目 | 内容 |
|---|---|
| 構築種別 | 完全新規構築（既存サイト・URL資産なし） |
| 技術スタック | Astro（コード・構造）+ microCMS（コンテンツ） ハイブリッド構成 |
| ホスティング | Xserver（ドメイン取得済み・サーバー未契約） |
| 運用体制 | 社内スタッフがAIを使いながらコンテンツ更新／Claude Codeがコード・構造を改善 |
| リポジトリ | GitHub（Private推奨） |

詳細な決定理由・経緯は [decisions.md](./decisions.md) を参照。

## 2. ページ一覧
確定サイトマップは [02-site-structure.md](./02-site-structure.md) を参照。

## 3. 機能要件

- レスポンシブ対応（モバイルファースト）
- お問い合わせフォーム（送信先・自動返信・スパム対策は未確定 → 7章参照）
- OGP設定（全ページ）
- パンくずリスト（構造化データ対応）
- 構造化データ: LocalBusiness / BreadcrumbList / Article / FAQPage / Service
- ブログ（カテゴリ・タグ対応）
- お知らせ
- XML Sitemap自動生成
- robots.txt
- 404ページ
- リダイレクト対応（将来のURL変更に備える）
- サイト内検索（将来実装・Pagefind想定、初期リリース対象外）

## 4. 非機能要件

| 分類 | 目標 |
|---|---|
| パフォーマンス | Core Web Vitals: LCP < 2.5s / INP < 200ms / CLS < 0.1 |
| アクセシビリティ | WCAG 2.1 AA準拠を目標 |
| セキュリティ | 静的サイト構成による攻撃面の最小化。フォームのスパム対策必須（reCAPTCHA等） |
| 保守性 | コード変更はPRベース。コンテンツ更新はGit知識不要（microCMS経由） |
| SEO | 全ページにtitle/meta/OGP設定。構造化データ実装。Search Console連携 |

## 5. 未確定事項（要確認・ブロッカー）

デザイン・コンテンツ制作に入る前に確定が必要な項目。

- [ ] 料金ページ：具体的な金額を出すか、「要相談」ベースにするか
- [ ] ロゴ・コーポレートカラー・指定フォント・猫モチーフ素材の有無
- [ ] 制作実績：掲載可能件数、守秘義務の制約有無
- [ ] お客様の声・レビューの収集有無
- [ ] 公開目標日・予算感
- [ ] LINE公式・Lステップの友だち追加導線の要否
- [ ] 問い合わせフォームの送信先・自動返信要否
- [ ] GA4 / Search Console / Microsoft Clarity等の計測要件
- [ ] Xserverプランの確定（SSH対応可否の確認）
- [ ] microCMSのプラン（コンテンツモデル数と料金の兼ね合いを事前確認）

## 6. 対象外（今回のスコープ外）
- 多言語対応（商圏・ターゲットから不要と判断。将来検討）
- ECカート機能
- 会員機能

## 7. フォーム要件（詳細化が必要）
現時点で未確定。以下を決めた上で実装設計を行う。
- 送信先メールアドレス（複数宛先か）
- 自動返信メールの要否・文面
- 個人情報の取り扱い同意チェックボックスの要否
- スパム対策（reCAPTCHA v3推奨）
