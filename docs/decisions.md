# 決定事項ログ（Decision Log）

プロジェクトの重要な意思決定を時系列で記録する。以後の意思決定もここに追記していく。新しい決定は末尾に追加し、番号は連番とする。

## D001: 技術スタックをAstro + microCMSハイブリッドに決定
- **日付**: 2026-07-04
- **背景**: Project.mdは「GitHub管理／Markdown管理／Claude Code対応／AI更新しやすい構成」を求める一方、ヒアリングの結果「実績・ブログを充実させたい」「社内スタッフがAIを使いながら更新する」という運用体制が判明した。
- **検討した代替案**:
  - Astro単体 + Markdown（Git） — Claude Code運用には最適だが、非技術スタッフがGit/PR操作を行うのは現実的でない
  - WordPress — スタッフには使いやすいが、DBベースでGit/AIエージェント運用と相性が悪い
  - Astro + WordPress（ヘッドレスCMS化） — 実現可能だが初期構築が複雑
- **決定**: コード・構造（デザイン/機能/SEOロジック）はAstro+Gitで管理しClaude Codeが改修する。頻繁に更新するコンテンツ（ブログ/実績/お知らせ/FAQ）はmicroCMSでスタッフが直接編集する。
- **トレードオフ**: コンテンツ本文はGit管理外（microCMSのクラウドDB）になる。

## D009: ドメイン・プラン詳細を確定、microCMS無料プランの適合性を検証
- **日付**: 2026-07-06
- **内容**:
  - ドメイン: **https://mikeneko.design**（設定済み）
  - Xserverプラン: **スタンダード**（既存契約）
  - microCMSプラン: **無料プラン（Hobby）**
- **検証結果**:
  - microCMS公式料金ページを確認した結果、無料プランはAPI数上限5個・コンテンツ数上限10,000件・APIコール数無制限。本設計の4コンテンツモデル（blog/works/news/faq）は問題なく収まり、低予算方針と矛盾しないことを確認した（[04-tech-architecture.md](./04-tech-architecture.md)）。ただし1メディア40MBのサイズ上限、独自ドメイン非対応（Enterprise限定）という制約はある
  - Xserverのスタンダードプランでのssh接続対応可否は、公式サイト上でプラン別の明確な対応表が見つからず未確認。ユーザー自身のサーバーパネルでの確認を依頼した。SSH利用不可の場合はFTP/SFTPデプロイにフォールバックする方針とした（[07-deploy-strategy.md](./07-deploy-strategy.md)）
- **影響**: インフラ面の確認事項はSSH対応可否のみとなり、それ以外はPhase2（Astro実装）着手の準備が整った。
