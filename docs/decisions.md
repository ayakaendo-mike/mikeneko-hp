# 決定事項ログ（Decision Log）

プロジェクトの重要な意思決定を時系列で記録する。以後の意思決定もここに追記していく。新しい決定は末尾に追加し、番号は連番とする。

---

## D001: 技術スタックをAstro + microCMSハイブリッドに決定
- **日付**: 2026-07-04
- **背景**: Project.mdは「GitHub管理／Markdown管理／Claude Code対応／AI更新しやすい構成」を求める一方、ヒアリングの結果「実績・ブログを充実させたい」「社内スタッフがAIを使いながら更新する」という運用体制が判明した。
- **検討した代替案**:
  - Astro単体 + Markdown（Git） — Claude Code運用には最適だが、非技術スタッフがGit/PR操作を行うのは現実的でない
  - WordPress — スタッフには使いやすいが、DBベースでGit/AIエージェント運用と相性が悪い
  - Astro + WordPress（ヘッドレスCMS化） — 実現可能だが初期構築が複雑
- **決定**: コード・構造（デザイン/機能/SEOロジック）はAstro+Gitで管理しClaude Codeが改修する。頻繁に更新するコンテンツ（ブログ/実績/お知らせ/FAQ）はmicroCMSでスタッフが直接編集する。
- **トレードオフ**: コンテンツ本文はGit管理外（microCMSのクラウドDB）になる。

## D002: 完全新規構築
- **日付**: 2026-07-04
- **内容**: 既存サイト・URL資産なし。ゼロからIA/URL設計を行う（301リダイレクト設計は不要）。

## D003: Xserver — ドメインのみ取得済み
- **日付**: 2026-07-04
- **内容**: サーバーは未契約。自動デプロイを見据えSSH対応プランを推奨（契約前に要確認）。

## D004: 運用体制はスタッフ+AI、2トラックのAI運用フローを採用
- **日付**: 2026-07-04
- **内容**:
  - コンテンツ更新: スタッフがAIで下書き → microCMSで公開（Git不要）
  - コード/構造改善: Claude CodeがPRを作成 → 人間承認（mainへの直push禁止）

## D005: サイトIAを確定
- **日付**: 2026-07-04
- **内容**: 会社概要と理念を統合。サービスをハブ+個別ページ化（`/service/xxx/`）。`/voice/`（お客様の声）と`/area/`（対応エリア）を追加提案し採用。地域名ごとのページ量産は行わない方針とした。

## D006: デザイン・料金・実績件数・予算スケジュールを確定
- **日付**: 2026-07-05
- **背景**: `corporate/営業資料/みけねこデザイン_会社案内.html` を確認したところ、暖色系（オレンジ〜イエロー）のカラーパレット・Google Fonts構成・簡易的な猫マークが既に存在することが判明した。
- **決定内容**:
  - **カラー/フォント**: 営業資料のカラーパレット（メイン: `#F58A1F`、サブ: `#FFC53D` 等）とフォント構成（Noto Sans JP / Outfit / IBM Plex Mono）をそのままコーポレートサイトに採用する（[08-design-guide.md](./08-design-guide.md)）
  - **ロゴ**: 営業資料内の簡易猫マークSVGを正式採用するか、新規制作するかは未確定のまま継続検討とする
  - **料金ページ**: 具体的な個別見積もりではなく、営業資料と同様の「大まかな価格帯」を仮掲載する。将来的にブラッシュアップ予定のため、現時点では仮の内容で公開してよい（[01-requirements.md](./01-requirements.md) 8章）
  - **実績掲載件数**: Web制作4〜5件、グラフィックデザイン4〜5点、名刺制作5〜6点、サポート事例2件。この4カテゴリをmicroCMSの`works.workType`として管理する（[04-tech-architecture.md](./04-tech-architecture.md)）
  - **公開目標・予算**: 2026年7月中に公開。予算はサーバー代＋AI利用料程度に抑える低予算方針とする
- **派生する追加検討事項**:
  - 低予算方針のため、microCMS・Xserverともに無料/最安プランで要件を満たせるか事前検証が必要（[04-tech-architecture.md](./04-tech-architecture.md) の注記参照）
  - 実際のサービス構成（Web制作／Web改善・リニューアル／ITサポート／デザイン制作／マーケティング支援／AI活用支援）が営業資料に既にあり、従来のサービスページ案に無かった「デザイン制作」「ITサポート」を`/service/`に追加した（[02-site-structure.md](./02-site-structure.md)）
