# SEO設計・コンテンツ戦略

## 1. キーワード設計
サービス軸 × 地域軸のマトリクスで、サービス個別ページを「ピラー」、ブログ記事を「クラスター」とするピラー&クラスター構造を採用する。

- **サービス軸**: ホームページ制作／WordPress制作／LP制作／LINE構築／AI導入支援／DX支援／保守運用・SEO改善
- **地域軸**: 松戸市／鎌ケ谷市／船橋市／市川市／千葉県

サービス個別ページ（`/service/xxx/`）がピラーページとして各サービスの中心キーワードを担い、ブログ記事がロングテール・関連質問キーワードを担って内部リンクでピラーページに集約する。

## 2. 構造化データ（Schema.org）

| ページ | 実装するschema |
|---|---|
| TOP／会社概要 | LocalBusiness, Organization |
| 全ページ | BreadcrumbList |
| ブログ記事 | Article |
| よくある質問 | FAQPage |
| サービス個別ページ | Service |

## 3. 地域SEOの方針
- 地域名ごとのドアウェイページ量産は行わない（Googleガイドライン違反リスク）
- `/area/` 1ページに対応エリアを集約し、Googleビジネスプロフィールと連携
- 地域名はブログ記事内で自然な文脈として扱う

## 4. E-E-A-T対策
- ブログ記事に執筆者情報を明示（誰が書いたか）
- 代表者の実績・資格・所属団体（商工会等）を会社概要ページに明示
- **AI生成記事であっても、公開前に必ず人間が事実確認・専門知見の追加・校正を行う**（生成AIの下書きをそのまま公開しない）

## 5. 技術的SEO
- Core Web Vitals目標値の維持（[01-requirements.md](./01-requirements.md) 参照）
- 画像はnext-gen形式（WebP/AVIF）を優先
- 全ページにcanonical設定
- XML Sitemap自動生成 → Google Search Console登録
- OGP・Twitter Card設定を全ページに実装

## 6. コンテンツ戦略

### コンテンツ種別
1. サービス解説（ピラーページ、更新頻度低）
2. ハウツー記事（ブログ、クラスター）
3. 事例紹介（`/works/` の実績とブログ記事をセットで作成し、案件完了ごとに両方追加）
4. 比較・選び方系記事（ブログ）

### AIの役割分担（重要）
- **下書き・構成案の作成**: AI（ChatGPT/Claude等）が担当
- **事実確認・専門知見の追加・最終校正**: 人間（スタッフ）が必ず担当

この分業を徹底しないと「AIが書いた薄い記事」がそのまま公開され、SEO上逆効果になるリスクがある。詳細な運用手順は [06-operation-guide.md](./06-operation-guide.md) を参照。

### 公開頻度
小規模体制であることを踏まえ、**月2本からスタート**し、運用が乗ってきたら本数を増やす。実績ページは案件完了のタイミングで都度追加。

### 実績とブログの連携
案件完了ごとに `/works/` の実績ページと関連するブログ記事をセットで作成する運用にすると、コンテンツが自然に積み上がる。
