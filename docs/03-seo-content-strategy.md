# SEO設計・コンテンツ戦略

## 1. キーワード設計
サービス軸 × 地域軸のマトリクスで、サービス個別ページを「ピラー」、ブログ記事を「クラスター」とするピラー&クラスター構造を採用する。

- **サービス軸**: ホームページ制作／WordPress制作／LP制作／LINE構築／AI導入支援／DX支援／保守運用・SEO改善／デザイン制作／ITサポート
- **地域軸（優先度つき）**:
  - **最優先**: 松戸市／鎌ケ谷市（自社の拠点かつ競合が弱く、地域SEO×AIOで上位を狙いやすい主戦場）
  - **準優先**: 東葛エリア（柏市／流山市／我孫子市／野田市）＋船橋市／市川市
  - **広域**: 千葉県／全国（オンライン対応）

サービス個別ページ（`/service/xxx/`）がピラーページとして各サービスの中心キーワードを担い、ブログ記事がロングテール・関連質問キーワードを担って内部リンクでピラーページに集約する。

> 商圏は「オンラインで全国対応」だが、SEO/AIO上の主戦場は東葛（松戸・鎌ケ谷中心）と明確に定める。詳細は3章。

## 2. 構造化データ（Schema.org）

|---|---|

## 3. 東葛ローカルSEO × AIO戦略（重要）

商圏はオンラインで全国対応だが、**検索とAI検索での主戦場は東葛エリア（松戸・鎌ケ谷中心）**に定める。競合が弱く、少ない労力で上位・AI引用を取りやすいため。

### 3-1. 基本方針「人には全国、検索エンジンとAIには地域も」
- **見た目・キャッチコピー**: 全国対応をうたって商圏を狭めない
- **titleタグ／会社概要／`/area/`／構造化データ**: 地域を明確に打ち出す
- 語順は必ず「**地域が先、全国があと**」。基本フレーズを全ページで統一する:
  > **松戸・鎌ケ谷など東葛地域を中心に、オンラインで全国対応**

### 3-2. 順位・AI引用を動かすレバー（影響が大きい順）
1. **Googleビジネスプロフィール**（地域検索・地図の最大要因）※登録状況・住所は要確認（[decisions.md#d012](./decisions.md#d012)）
2. **titleタグ**に地域＋サービスを入れる（下の雛形参照）
3. **ページ内H1・本文**に地域名を自然に含める（TOPのヒーロー/ABOUT、`/area/`）
4. **構造化データ** LocalBusinessの`areaServed`に市区を明示
5. **地域からの被リンク・引用**（商工会／青年会議所など。営業資料に鎌ケ谷JC・日本JCとの関わりあり＝東葛では有効）
6. **ブログのロングテール**（「松戸市の中小企業が〜」等）

> meta descriptionは順位の直接要因ではない（スニペット表示とクリック率に効く）。地域名は入れるが、主力はtitle・本文・構造化データ・Googleビジネスプロフィールに置く。

### 3-3. title / meta description 雛形
- TOP（title）: `松戸・鎌ケ谷のホームページ制作／IT・AI活用支援｜みけねこデザイン`
- TOP（description）: `千葉県松戸市・鎌ケ谷市など東葛地域を中心に、オンラインで全国対応。ホームページ制作からWeb改善・IT・AI活用・デザイン制作まで、事業成長を伴走するITパートナーです。`
- サービス個別（例）: `松戸市のホームページ制作｜みけねこデザイン`
- 会社概要（本文の自己定義・AIO用の一文）: `みけねこデザインは、千葉県松戸市・鎌ケ谷市を中心に東葛エリアで活動する、Web制作・IT・AI活用支援の会社です。オンラインで全国対応。`

### 3-4. `/area/` ページ（地域SEOの本命受け皿）
- 対応エリア（松戸・鎌ケ谷を筆頭に東葛各市→千葉県→全国オンライン）を1ページに集約
- **地域名ごとのドアウェイページ量産はしない**（Googleガイドライン違反リスク）。1ページ集約＋ブログでの自然言及に留める
- Googleビジネスプロフィールへの導線を設置
- 「なぜ東葛の企業に伴走できるのか（地元の商習慣・団体との関わり）」を書きE-E-A-Tを補強

### 3-5. AIO（AI検索最適化）
ChatGPT/Perplexity/GoogleのAI概要は「何者か・どこの会社か」が明快なサイトを引用しやすい。
- 会社概要に**一文の明快な自己定義**（3-3参照）を置く
- **FAQ**を厚くする（例:「松戸市でホームページ制作を頼めますか？」→明確な回答）。FAQPage構造化データと一致させAIに拾わせる
- 第三者サイト（商工会・JC・地域メディア）での言及＝AIにとっての裏取りになるため、掲載機会を活用する

## 4. E-E-A-T対策
- ブログ記事に執筆者情報を明示（誰が書いたか）
- 代表者の実績・資格・所属団体（商工会等）を会社概要ページに明示
- **AI生成記事であっても、公開前に必ず人間が事実確認・専門知見の追加・校正を行う**（生成AIの下書きをそのまま公開しない）

## 5. 技術的SEO
- Core Web Vitals目標値の維持（[01-requirements.md](./01-requirements.md) 参照）
- 画像はnext-gen形式（WebP/AVIF）を優先
- 全ページにcanonical設定
- XML Sitemap自動生成 → Google Search Console登録
- OGP・Twitter Card設定を全ページに実装

## 6. コンテンツ戦略

### コンテンツ種別
1. サービス解説（ピラーページ、更新頻度低）
2. ハウツー記事（ブログ、クラスター）
3. 事例紹介（`/works/` の実績とブログ記事をセットで作成し、案件完了ごとに両方追加）
4. 比較・選び方系記事（ブログ）

### AIの役割分担（重要）
- **下書き・構成案の作成**: AI（ChatGPT/Claude等）が担当
- **事実確認・専門知見の追加・最終校正**: 人間（スタッフ）が必ず担当

この分業を徹底しないと「AIが書いた薄い記事」がそのまま公開され、SEO上逆効果になるリスクがある。詳細な運用手順は [06-operation-guide.md](./06-operation-guide.md) を参照。

### 公開頻度
小規模体制であることを踏まえ、**月2本からスタート**し、運用が乗ってきたら本数を増やす。実績ページは案件完了のタイミングで都度追加。

### 実績とブログの連携
案件完了ごとに `/works/` の実績ページと関連するブログ記事をセットで作成する運用にすると、コンテンツが自然に積み上がる。

## 7. ブランドメッセージ・トーン（既存営業資料より）

既存の営業資料（[08-design-guide.md](./08-design-guide.md) 参照）に、そのまま使える一貫したメッセージングが既にあったため、サイトのコピーライティングもこれに揃える。

- **キャッチコピー**: 「Webを、事業の『働く資産』へ。」
- **ポジショニング**: 「社外のWeb担当」「つくって、終わりにしない」／**「制作会社」ではなく「事業成長のパートナー（ITパートナー）」**（[decisions.md#d011](./decisions.md#d011)。会社紹介文で「制作会社」と名乗らない）
- **サイクル**: MAKE（制作）→ LAUNCH（公開）→ OPERATE（運用）→ IMPROVE（改善）
- **特徴（4つ／見出しは「みけねこデザインの特徴」）**:
  1. 伴走型サポート — 公開後も「社外Web担当」として運用まで並走
  2. ワンストップ対応 — Web・グラフィック・ITの窓口を一本化
  3. 数字にもとづく改善提案 — アクセス解析データで判断
  4. AIを味方にする制作 — AI活用でスピードと品質を両立

TOP/会社概要ページのコピーおよびブログのトーンは、この4つの特徴を軸に一貫させる。AIが下書きを作成する際も、このメッセージングと3章の地域基本フレーズをプロンプトに含めることで表記ゆれを防ぐ（[06-operation-guide.md](./06-operation-guide.md) に追記予定）。
