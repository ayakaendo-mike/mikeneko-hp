# サイト構成・IA・URL設計

## 設計方針
- 会社概要と理念は統合（理念単体でのSEO需要は薄く、ストーリーとして繋げた方がUX上強い）
- サービスはハブページ＋個別ページに分割し、複合キーワード（例:「松戸市 ホームページ制作」）ごとに最適化
- 地域名ごとのページ量産（ドアウェイページ）は行わない。対応エリアは1ページに集約し、Googleガイドライン違反リスクを回避
- 各ページのコンテンツ管理元（Astroコード側 / microCMS側）を明記し、更新担当者が迷わないようにする

## ブログとお知らせの役割分離
- **ブログ**: SEO記事。ハウツー・比較・事例紹介などピラー/クラスター構造でロングテール流入を狙う
- **お知らせ**: 会社の一次情報（お休み案内、サービス改定、メディア掲載等）。SEO目的ではなく信頼性・鮮度の担保が目的

## 地域SEOの扱い
- `/area/` 1ページに集約し、対応地域・Googleビジネスプロフィールへの導線をまとめる
- 地域名ごとのランディングページ量産は行わない
- ブログ記事内で自然な文脈として地域名を扱う（例:「松戸市の製造業がホームページをリニューアルする際の注意点」）

## LINE公式導線（将来対応）
Lステップは使わず、LINE公式（+エルメ）をテスト運用中。初期リリースのスコープには含めないが、`/contact/`や`/service/line/`に友だち追加ボタンを設置できるよう、コンポーネント設計上は考慮しておく（[decisions.md#d007](./decisions.md#d007)）。
